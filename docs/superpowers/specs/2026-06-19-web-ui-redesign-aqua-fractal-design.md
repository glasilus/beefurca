# Beefurca Web — переработка UI: «Aqua × Fractal»

Дата: 2026-06-19
Область: `apps/web` (mobile НЕ трогаем). Полная переработка визуального языка
и рефакторинг дизайн-системы. Бэкенд-логика не меняется.

---

## 1. Тезис и метафора

Beefurca — платформа организации и учёта соревнований по **любой** дисциплине
(шахматы, CS, настольный теннис — что угодно). Аудитория: организаторы, игроки,
судьи, админы. Задача интерфейса — **управлять соревнованием и наблюдать за ним**.

Визуальный тезис: **инженерный инструмент в стеклянном корпусе Aqua, где
единственный источник цвета и «жизни» — живая алгоритмическая (фрактальная)
графика, заключённая в спокойный стеклянно-алюминиевый хром.**

Это НЕ киберпанк и НЕ «типичный сайт». Узнаваемый ранний macOS X (Aqua) +
Y2K-скевоморфизм, переосмысленный аккуратно. Уходим от текущего «дешёвого/
колхозного» вайба, источники которого: неоновый градиент `красный→фиолетовый→
жёлтый` (`activeGrad`) и пиксельный шрифт `Beast`.

**ПРИОРИТЕТ — мгновенная узнаваемость macOS.** Стиль должен читаться как
«старый Mac OS X» без объяснений. Главные носители узнаваемости (важнее всего
остального): (1) фон **Aqua Aurora** — фирменный «шёлковый» переливающийся
сине-градиентный фон-обои Mac OS X; (2) **полупрозрачные матовые панели** поверх
него (окна Aqua «парят» над обоями); (3) brushed-metal заголовки + светофор-точки;
(4) глянцевые gel-кнопки; (5) скруглённые верхние углы окон и видимый pinstripe.

Референсы (папка `рефы/`): macOS X Aqua (ref 2, 4 — brushed metal, Aurora,
светофор-кнопки, леденцовые кнопки), фрактальная графика (тёмная — Мандельброт,
светлая — органические процедурные формы, ref 5), алгоритмическая «чип-разводка»
(photo_1..6) как декоративный слой.

### 2.3 Фон Aqua Aurora (ядро узнаваемости)

Фон приложения — не плоская заливка, а **живой aurora-градиент** (фирменные обои
Mac OS X). Реализация лёгкая и GPU-дружелюбная: несколько наложенных
`radial-gradient`/`conic-gradient` + один медленно смещающийся слой (CSS
`transform`/`background-position`, ~30–60s, выключается при `prefers-reduced-motion`).
Без тяжёлых canvas-циклов в фоне.

- **Светлая «Aqua Clarity»:** шёлковые голубые ленты (`#DCE9FF`, `#A9CBF0`,
  `#6FA8E6`) с белыми сполохами — как классические обои Aqua.
- **Тёмная «Aqua Obsidian»:** глубокий полуночный aurora (`#0A1730`, `#102A4D`,
  `#06121F`) со светящейся сине-циановой лентой. Тёмная тема НЕ должна быть
  плоским графитом — это была главная претензия.

Панели/карточки — **полупрозрачные** (`backdrop-filter: blur`, фон
`color-mix(panel, transparent)`), чтобы aurora мягко просвечивала и «макос»
читался сразу. Контент при этом остаётся контрастным (панель достаточно плотная).

### 2.4 Акцент и статусы (обе темы)

- **Акцент Aqua-gel (голубой):** блик `#8FC0FF` → ядро `#2E86F0` → низ `#0A5BD0`.
  Используется для главных действий, активных связей, выделения.
- **Статусы = светофор macOS (часть подписи):**
  - `--status-live` `#FEBC2E` (янтарь) — идущий матч/турнир.
  - `--status-done` `#18B5C7` (ртутный циан) — завершено / победный путь.
  - `--status-win` `#28C840` (зелёный) — победа/подтверждено.
  - `--status-danger` `#FF5F57` (красный) — отклонение/удаление/ошибка.

Контраст текста к фону — не ниже WCAG AA 4.5:1 на основном тексте.

## 3. Типографика — «Aqua × Algorithm»

Три роли, сквозная **нативная кириллица**, без запрещённых шрифтов
(Inter / JetBrains Mono / прочие AI-дефолты — запрещены).

| Роль | Гарнитура | Применение | Начертания |
|---|---|---|---|
| **Display** | **Unbounded** | логотип, hero, крупный живой счёт, H1 в hero | Bold / Black |
| **Текст/UI** | **Fira Sans** | заголовки H1–H3, body, кнопки, формы | Light→Heavy |
| **Узкий/подписи** | **Fira Sans Condensed** | заголовки таблиц, eyebrow-лейблы (caps, +0.06em) | Medium |
| **Данные** | **IBM Plex Mono** | ELO, счёт в таблицах, таймстампы, ID/хэши | Regular/Bold |

Обоснование: `Unbounded` — геометрический монолинейный, узнаваемый и bold,
«вычисленные» кривые рифмуются с фрактальной/процедурной графикой и gel-глянцем
(закрывает «ретро-узнаваемый» и «алгоритмический» векторы сразу), родная
кириллица. `Fira Sans` — тёплый гуманист, прямая линия Lucida Grande («старый
macOS X»), кириллица команды Spiekermann. `IBM Plex Mono` — техничный ретро-инженерный
моно с отличной кириллицей.

Тип-шкала (rem, base 16px):
- Display hero: 44–56px / 1.0 (Unbounded Black)
- H1: 28px / 1.2 (Fira Sans Bold)
- H2: 20px / 1.3 (Fira Sans Bold/Medium)
- H3: 16px / 1.35 (Fira Sans Medium)
- Body: 14px / 1.5 (Fira Sans Regular)
- Caption/Table header: 12px / 1.1 caps +0.06em (Fira Sans Condensed Medium)
- Data: 13px / 1.0 (IBM Plex Mono)

Удаляются: `Beast` (пиксель), `Grafmassa`, `Unifix SP`.

## 4. Геометрия и глянец (баланс с уклоном в ретро-Aqua)

- **Радиусы:** контролы 5px, карточки/окна 8px (**верхние углы окон скруглены
  сильнее**, ~10px — характерная форма окна Aqua), **главная кнопка — pill**
  (полное скругление) с настоящим Aqua-бликом сверху. Без острых 0px и без
  «капсул везде».
- **Глубина — скевоморфная, многослойная** (из spec): outset для выпуклых
  элементов (верхний внутренний блик `inset 0 1px 0 rgba(255,255,255,…)` +
  нижняя внешняя тень), inset для вогнутых (поля ввода, JSONB-плашки).
- **Окна-панели Aqua:** brushed-metal заголовок (градиент `--chrome-*`) + три
  «светофорные» точки. Точки **функциональны** — кодируют состояние турнира
  (янтарь = идёт, зелёный = завершён, серый = черновик/draft). Это оправданный
  риск: узнаваемо, играбельно, несёт смысл (подтверждено пользователем).
- **Fluted glass** (рифлёное матовое стекло, `backdrop-filter: blur(16px)` +
  вертикальные микрополосы) — модалки, тосты, выпадающие списки, оверлеи.
- **Pinstripe** — заметные (но негромкие) горизонтальные полосы на панелях и в
  brushed-metal заголовках (узнаваемая Aqua-текстура).
- **Дизеринг НЕ используется** — фракталы самодостаточны и хорошо выглядят без
  шума (по фидбеку). Из spec/реализации убирается.

Избегаем «нейрослоп»-приёмов: никаких cream+serif+terracotta, near-black+acid,
broadsheet-hairlines; никаких дефолтных шрифтов; глянец — намеренный и
дисциплинированный, не «по всему экрану».

## 5. Подпись (signature) — фрактальный медальон под стеклом Aqua

Единственный «громкий» элемент, остальное вокруг — тихо. Каждый турнир /
дисциплина / игрок получает процедурный фрактал (Julia/Mandelbrot из хэша ID),
вставленный в **глянцевую gel-линзу Aqua**. Один и тот же приём проходит насквозь:

- **Логотип** — фрактал внутри знака «B» (не плоский голубой), под gel-стеклом.
  Словесная часть — Unbounded.
- **Аватары** турниров/игроков/дисциплин (`FractalAvatar` в gel-оправе).
- **Судейская пломба** (`FractalSeal`) — фрактал «вдавливается» как восковая
  печать при фиксации счёта.
- **Узлы турнирной сетки** (`BracketCanvas`) — крупные медальоны участников.
- **Амбиентный фон** — Aqua Aurora (раздел 2.3).

«Алгоритмическая красота в стекле» = бренд.

### 5.0 Расширенная роль фракталов (утверждено)

Фракталы — не только подпись, но и **рабочий материал** интерфейса:
- **Замена изображений** (вместо S3/загрузки картинок): аватары турниров, игроков,
  дисциплин, команд; обложки/баннеры турниров; пустые состояния. Любое место, где
  «нужна картинка» — генерируем фрактал из ID/хэша.
- **Текстуры и акценты** — где уместно: декоративные подложки секций, hero-баннеры,
  углы карточек, заглушки. Не перегружать контентные зоны (читабельность важнее).
- **Анимации** — лёгкие и уместные: плавная «закрутка» пломбы при фиксации счёта,
  мягкое дыхание/параллакс фрактального баннера, появление медальонов. Всегда с
  учётом `prefers-reduced-motion`.

Окрашивание фракталов — как в утверждённой витрине (`docs/design-preview.html`):
генератор Жюлиа из первой версии, яркая многоцветная палитра (hue-span ~330–360,
sat ~0.95), без дизеринга. Логотип — отдельные параметры `c=0.285+0.01i`, радужный
спектр (`hue=0`, `span=360`, поднятая яркость).

### 5.0.1 Оптимизация генеративного слоя (обязательно)

Производительность критична — фракталов будет много:
- Рендер в **Web Worker** (вне UI-потока) + `OffscreenCanvas` где доступно;
  результат — в `<canvas>` или как `toDataURL`/blob для `<img>`.
- **Кэширование**: один и тот же seed/размер не пересчитывать (in-memory + при
  возможности `sessionStorage`/`caches`). Детерминированный рендер по хэшу.
- Рендер по размеру элемента (маленькие аватары — маленький canvas, не 1:1 с DPR
  без нужды); ограничить `maxIter` под размер.
- Ленивость: рендер при появлении в вьюпорте (`IntersectionObserver`).
- Анимации — только `transform`/`opacity` (GPU), без layout-thrash; пауза вне
  вьюпорта; полное отключение при `prefers-reduced-motion`.
- Aurora-фон — чистый CSS (без постоянного canvas-цикла).

### 5.1 Турнирная сетка — «энергодерево» (переделать, текущий вид скучный)

Сетка переосмысливается как живой топологический граф, а не список карточек:
- **Узлы** — окна-матчи Aqua с крупными фрактал-медальонами участников и счётом
  (IBM Plex Mono).
- **Связи** — объёмные изогнутые **SVG-кривые** (не прямые палки): неактивные —
  нейтральные hairline; активный путь пульсирует янтарём (live); победный путь
  плавно окрашивается в ртутный циан (done), ведя взгляд к финалу.
- **Холст** — поверх Aqua Aurora; pan/zoom сохраняется.
- Разные форматы (Single/Double Elim, Round Robin, Swiss) сохраняют свою
  визуализацию из `BracketCanvas`, но в новом языке (медальоны + gel + кривые).

## 6. Архитектура рефакторинга (по-грамотному, не латками)

### 6.1 Слой токенов
Семантические CSS-переменные (раздел 2) в `globals.css` для `:root` (light) и
`.dark`. Маппинг в `tailwind.config.ts`: `surface/panel/panel-sunken/chrome/
border/hairline/text/text-muted/accent/status-*`.
**Удаляются костыли:** инверсия `html:not(.dark) .text-slate-*`, переопределения
`bg-slate-800/900`, тема-зависимый `component-card-dark`. Это и есть источник
«белого на белом» и нечитаемых надписей.

### 6.2 Примитивы (новые компоненты в `apps/web/src/components/ui/`)
Каждый — изолированный, с понятным интерфейсом, заменяет ad-hoc разметку:
- `Button` — варианты `gel` (главная, glossy pill) / `secondary` / `ghost` /
  `danger`; размеры; состояния loading/disabled; видимый focus-ring.
- `Window` / `Card` — Aqua-окно: brushed-metal титлбар, опц. светофор-статус,
  слот контента; вариант «бесшовная карточка» без титлбара.
- `Field` / `Input` / `Select` / `Checkbox` — inset-плашки, лейблы, ошибки.
- `Badge` — статус-пилюля по `status-*` (live/done/win/danger/draft).
- `Modal` — fluted glass, фокус-трэп, Esc, опирается на существующий `useConfirm`.
- `Table` — Fira Sans Condensed заголовки, IBM Plex Mono данные, зебра по hairline.
- `Tabs`, `EmptyState` (фрактал + призыв к действию), `PageHeader`.

`Toast` / `ConfirmDialog` / `ThemeToggle` / `Nav` — переводятся на токены и
примитивы (логика сохраняется).

### 6.3 Генеративный слой
`FractalAvatar` (gel-оправа, без дизеринга), `FractalSeal`, фрактал-логотип,
узлы/связи `BracketCanvas` (статусные цвета, SVG-кривые), фон Aqua Aurora.
Цвет фракталов — яркий и многоцветный (контраст с тихим хромом).

### 6.4 Типографика
`@font-face` для Unbounded / Fira Sans / Fira Sans Condensed / IBM Plex Mono
(self-host в `public/fonts`, `font-display: swap`), тип-шкала, утилиты Tailwind.

### 6.5 Прогон страниц (все 11 маршрутов)
`page` (лендинг), `login`, `dashboard`, `tournaments` (список), `tournaments/[id]`,
`tournaments/create`, `admin`, `disciplines`, профиль, `bracket`-вью и пр.
На каждой: перевод на примитивы и токены, точечная починка контраста
(белое-на-белом, нечитаемые надписи), удаление мёртвых стилей (`border-red-955`
и т.п.). Чекпоинты после фундамента и после ключевых страниц.

### 6.6 Тема по умолчанию
Тёмная остаётся по умолчанию (как сейчас), но светлая «Aqua Clarity» становится
по-настоящему аутентичной. Обе — first-class. Без вспышки при загрузке
(next-themes уже настроен).

## 7. Качественный пол (без анонсирования)
Адаптив до мобильного, видимый keyboard-focus, `prefers-reduced-motion`
уважается (aurora-анимация и пульсация связей отключаются), вес шрифтов
контролируется (subset кириллица+латиница где возможно).
**Эмодзи запрещены** — нигде в UI/коде. Вместо них иконки (phosphor) или SVG.
В существующем коде убрать `🔒 Приватный`, `★ официальная` и т.п.

## 8. Порядок работ
1. Фундамент: токены + типографика + примитивы + генеративный слой. **Чекпоинт.**
2. Ключевые страницы: лендинг, login, dashboard, tournaments/[id]. **Чекпоинт.**
3. Остальные страницы: список турниров, create, admin, disciplines, профиль.
4. Финальный проход: контраст-аудит обеих тем, адаптив, a11y, сборка
   `pnpm --filter web build` (зелёная, все маршруты).

## 9. Что НЕ входит
- Мобильное приложение (`apps/mobile`).
- Изменения бэкенд-логики/схемы БД.
- Сквозная типизация фронта (`any` → `@beefurca/shared-types`) — отдельная
  некритичная задача, не часть этой переработки.
